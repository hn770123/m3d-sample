# iPhone Safari向け 3Dレンダリングテスト（裸眼立体視）プロジェクト草案

## 1. プロジェクト概要
iPhoneのSafariブラウザ上で動作する、Three.jsを用いた3Dレンダリングのテストサンプルを作成し、GitHub Pagesで公開する。
特にスマートフォンの「縦画面（ポートレート）」を左右に分割し、視差を設定したオブジェクト（キューブ）を配置することで、裸眼での立体視（交差法または平行法）のテストを行うことを目的とする。

## 2. 技術スタック
ビルド環境を構築せず、シンプルに実装・公開できる構成を採用する。

* **3Dライブラリ:** Three.js (CDN経由で読み込み)
* **言語:** HTML5, CSS3, JavaScript (ES Modules形式で記述)
* **ホスティング:** GitHub Pages
* **バージョン管理:** Git / GitHub

## 3. 画面構成
* **レイアウト:** デバイスの縦画面（ポートレート表示）を中央で左右に2分割する。
* **カメラ設定:** 左右の画面それぞれに独立したカメラ（またはThree.jsのStereoCameraのカスタム設定）を配置し、わずかに水平方向に視差（オフセット）を持たせる。
* **表示オブジェクト:** 中央に配置された同一のキューブを、左右のカメラからレンダリングする。

## 4. 段階的テストシナリオ
実装とテストは以下のステップに分けて段階的に進める。

### Step 1: 自動アニメーションと視差の確認
* 左右に分割された画面に、ゆっくりと回転するキューブを表示する。
* 視差が正しく設定されており、裸眼で立体的に見えるかをテストする。
* デバイスの解像度（Retinaディスプレイ）に合わせたレンダリング設定（`setPixelRatio`）を行う。

### Step 2: タッチインタラクションの導入
* 画面へのタッチ操作（スワイプなど）を取得し、キューブの動作に反映させる。
* 例：ユーザーが画面をスワイプすると、回転方向が変わったり、回転速度が変化する。
* Safari特有のスクロール挙動（バウンスなど）を防ぐための、`touch-action: none` などのCSS設定やイベント制御をテストする。

### Step 3: （後日検討）
* さらに複雑なオブジェクトの描画、ライティングの変更、マテリアルのテストなどを必要に応じて追加していく。
